## Cursor AI란?

2026년 현재 개발자 사이에서 가장 빠르게 성장하고 있는 코드 에디터, **Cursor AI**. VS Code를 기반으로 만들어진 이 AI 네이티브 IDE는 코드 작성, 리팩토링, 디버깅, 문서화까지 AI가 직접 도와주는 차세대 개발 환경입니다.

GitHub Copilot이 자동완성에 집중했다면, Cursor는 **프로젝트 전체를 이해하고 대화하듯 코딩하는 것**에 초점을 맞춥니다. 2025년 말 기준 월간 활성 사용자 200만 명을 돌파하며 명실상부한 AI IDE 1위 자리를 차지하고 있습니다.

## 설치 및 초기 설정

### 다운로드

공식 사이트([cursor.com](https://cursor.com))에서 운영체제에 맞는 설치 파일을 다운로드합니다.

- **Windows**: `.exe` 설치 파일
- **macOS**: `.dmg` 패키지
- **Linux**: `.AppImage` 또는 `.deb`

### VS Code 설정 마이그레이션

이미 VS Code를 사용 중이라면 **원클릭 마이그레이션**으로 기존 환경을 그대로 가져올 수 있습니다.

- 확장 프로그램 자동 설치
- 키바인딩 및 설정 복사
- 테마 및 폰트 설정 유지

설치 후 첫 실행 시 "Import VS Code Settings" 옵션을 선택하면 됩니다.

## 핵심 기능 5가지

### 1. Tab 자동완성 (Copilot++)

일반적인 AI 자동완성을 넘어선 **다음 편집 예측(Next Edit Prediction)** 기능입니다.

- 현재 커서 위치의 다음 코드를 예측
- 여러 줄에 걸친 편집도 한 번에 제안
- 기존 코드 패턴을 학습하여 맥락에 맞는 제안

```typescript
// 함수 시그니처만 입력하면...
function calculateTotalPrice(items: CartItem[])

// Tab 한 번으로 전체 구현이 완성됩니다
function calculateTotalPrice(items: CartItem[]): number {
  return items.reduce((total, item) => {
    const discount = item.coupon ? item.price * item.coupon.rate : 0;
    return total + (item.price - discount) * item.quantity;
  }, 0);
}
```

### 2. Cmd+K 인라인 편집

코드 블록을 선택하고 `Cmd+K` (Windows: `Ctrl+K`)를 누르면 **자연어로 코드를 수정**할 수 있습니다.

**활용 예시:**
- "이 함수에 에러 핸들링 추가해줘"
- "TypeScript 타입을 더 엄격하게 변경해줘"
- "이 코드를 async/await 패턴으로 변환해줘"
- "변수명을 더 명확하게 바꿔줘"

선택한 코드만 정확히 수정하기 때문에 **부분 리팩토링**에 매우 유용합니다.

### 3. Cmd+L 채팅 (AI Chat)

사이드바에서 AI와 대화하며 코딩할 수 있는 기능입니다.

**주요 특징:**
- 현재 열린 파일의 컨텍스트를 자동으로 이해
- `@파일명`으로 특정 파일 참조 가능
- `@codebase`로 프로젝트 전체를 컨텍스트에 포함
- `@web`으로 웹 검색 결과를 포함한 답변 생성
- 코드 블록을 직접 적용하는 "Apply" 버튼

### 4. Composer (에이전트 모드)

Cursor의 가장 강력한 기능으로, **여러 파일을 동시에 생성하고 수정**하는 AI 에이전트입니다.

```
프롬프트 예시:
"사용자 인증 API를 만들어줘.
- /api/auth/login, /api/auth/register, /api/auth/logout 엔드포인트
- JWT 토큰 기반 인증
- Prisma ORM으로 User 모델 정의
- 에러 핸들링과 입력 검증 포함"
```

이렇게 요청하면 Composer가 **관련 파일들을 자동으로 생성하고 수정**합니다:
- `prisma/schema.prisma` - User 모델 추가
- `src/lib/auth.ts` - JWT 유틸리티
- `src/app/api/auth/login/route.ts` - 로그인 API
- `src/app/api/auth/register/route.ts` - 회원가입 API
- `src/app/api/auth/logout/route.ts` - 로그아웃 API

### 5. @Docs 커스텀 문서 참조

프로젝트에서 사용하는 라이브러리 문서를 Cursor에 등록하면, AI가 **최신 공식 문서를 참고하여 답변**합니다.

- `@Next.js` - Next.js 공식 문서 기반 답변
- `@Tailwind` - Tailwind CSS 클래스 추천
- `@Prisma` - Prisma 스키마 및 쿼리 가이드

설정 방법: `Cursor Settings > Features > Docs > Add new doc`에서 문서 URL을 등록합니다.

## VS Code vs Cursor 비교

기존 VS Code 사용자라면 가장 궁금한 질문: **"VS Code에서 Cursor로 갈아타야 할까?"**

| 비교 항목 | VS Code + Copilot | Cursor AI |
|----------|-------------------|-----------|
| **기반** | VS Code | VS Code Fork |
| **AI 모델** | GPT-4o | Claude, GPT-4o, 자체 모델 선택 |
| **자동완성** | 단일 라인 위주 | 다중 라인 + 편집 예측 |
| **컨텍스트** | 현재 파일 위주 | 프로젝트 전체 인덱싱 |
| **에이전트 모드** | ❌ (제한적) | ✅ Composer |
| **인라인 편집** | ❌ | ✅ Cmd+K |
| **확장 호환** | 네이티브 | VS Code 확장 대부분 호환 |
| **가격** | $10/월 (Copilot) | $20/월 (Pro) |
| **오프라인** | 부분 지원 | 기본 편집만 가능 |

**결론:** 단순 자동완성 이상의 AI 협업을 원한다면 Cursor, 기존 워크플로우 유지가 중요하다면 VS Code + Copilot이 좋은 선택입니다.

## 실전 활용 팁 7가지

### 팁 1. .cursorrules 파일로 프로젝트 규칙 설정

프로젝트 루트에 `.cursorrules` 파일을 만들면 AI가 프로젝트 컨벤션을 자동으로 따릅니다.

```
# .cursorrules 예시
- TypeScript strict 모드 사용
- 컴포넌트는 함수형으로만 작성
- API 응답은 반드시 try-catch로 감싸기
- CSS는 Tailwind CSS만 사용
- 변수명은 camelCase, 컴포넌트는 PascalCase
- 한국어 주석 사용
```

### 팁 2. @codebase를 적극 활용

채팅에서 `@codebase`를 사용하면 프로젝트 전체를 컨텍스트로 포함합니다.

```
@codebase 현재 프로젝트에서 사용 중인 인증 방식을 분석하고,
보안 취약점이 있다면 개선 방안을 제안해줘
```

이 기능은 **코드 리뷰, 리팩토링 계획, 버그 추적**에 특히 유용합니다.

### 팁 3. 이미지로 UI 구현하기

Cursor 채팅에 **디자인 스크린샷이나 목업 이미지**를 붙여넣으면 해당 UI를 코드로 구현해줍니다.

1. 디자인 이미지를 채팅창에 드래그 앤 드롭
2. "이 디자인을 React + Tailwind CSS로 구현해줘"라고 요청
3. 컴포넌트 코드가 자동 생성

Figma 스크린샷을 바로 코드로 변환하는 워크플로우가 가능합니다.

### 팁 4. 터미널 Cmd+K 활용

터미널에서 `Cmd+K` (Windows: `Ctrl+K`)를 누르면 **자연어로 터미널 명령어를 생성**할 수 있습니다.

- "3일 이내에 수정된 TypeScript 파일 목록 보여줘"
- "현재 프로젝트의 미사용 npm 패키지 찾아줘"
- "Docker 컨테이너 로그에서 에러만 필터링해줘"

복잡한 쉘 명령어를 외울 필요 없이 자연어로 요청하면 됩니다.

### 팁 5. 점진적 프롬프트 전략

한 번에 모든 것을 요청하지 말고, **단계적으로 요청**하면 더 좋은 결과를 얻습니다.

```
❌ 나쁜 예: "풀스택 이커머스 사이트 만들어줘"

✅ 좋은 예:
1단계: "상품 목록 API를 만들어줘. GET /api/products"
2단계: "이 API에 페이지네이션과 필터링을 추가해줘"
3단계: "상품 목록을 보여주는 React 컴포넌트를 만들어줘"
4단계: "이 컴포넌트에 무한 스크롤을 적용해줘"
```

### 팁 6. 코드 리뷰에 AI 활용

PR을 올리기 전에 Cursor로 셀프 코드 리뷰를 진행하세요.

```
@codebase 내가 오늘 수정한 파일들을 리뷰해줘.
- 버그 가능성이 있는 부분
- 성능 개선이 가능한 부분
- 타입 안전성이 부족한 부분
을 체크해줘
```

## 자주 묻는 질문 (FAQ)

### Q. Cursor는 VS Code 확장 프로그램과 호환되나요?
**네, 대부분 호환됩니다.** Cursor는 VS Code를 포크하여 만들었기 때문에 기존 확장 프로그램의 약 95% 이상이 정상 작동합니다. 다만 일부 VS Code 전용 기능에 의존하는 확장은 문제가 있을 수 있습니다.

### Q. 내 코드가 AI 학습에 사용되나요?
**Privacy Mode를 활성화하면 사용되지 않습니다.** `Settings > Privacy > Privacy Mode`를 켜면 코드가 AI 모델 학습에 사용되지 않습니다. Business 플랜은 기본으로 Privacy Mode가 활성화되어 있습니다.

### Q. 인터넷 없이도 사용 가능한가요?
**기본 편집은 가능하지만 AI 기능은 사용 불가합니다.** AI 기능은 클라우드 기반이므로 인터넷 연결이 필요합니다. 기본적인 코드 편집과 VS Code 기능은 오프라인에서도 작동합니다.

### Q. GitHub Copilot과 동시에 사용할 수 있나요?
기술적으로는 가능하지만 **추천하지 않습니다.** 두 AI가 동시에 자동완성을 시도하면 충돌이 발생할 수 있습니다. Cursor를 사용한다면 Copilot 확장은 비활성화하는 것이 좋습니다.

### Q. 무료 플랜으로도 충분한가요?
**학습과 개인 프로젝트에는 충분합니다.** 다만 본격적인 업무에 사용한다면 자동완성 제한(월 2,000회)이 금방 소진됩니다. 2주 이상 써보고 Pro 업그레이드를 결정하세요.

## 마무리

Cursor AI는 단순한 코드 에디터를 넘어 **AI 페어 프로그래머**에 가깝습니다. 프로젝트 전체를 이해하고, 여러 파일을 동시에 수정하며, 자연어로 대화하듯 코딩할 수 있는 환경은 확실히 생산성을 크게 높여줍니다.

물론 AI가 생성한 코드를 무조건 신뢰해서는 안 됩니다. **AI는 도구이고, 최종 판단은 개발자의 몫**입니다. 하지만 이 도구를 잘 활용하면 반복적인 작업에서 해방되어 더 창의적인 문제 해결에 집중할 수 있습니다.

지금 바로 Cursor를 설치하고 `Cmd+K`부터 시작해보세요. AI와 함께하는 개발 경험이 여러분의 코딩 습관을 완전히 바꿔놓을 것입니다.
