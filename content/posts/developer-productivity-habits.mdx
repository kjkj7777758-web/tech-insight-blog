## 들어가며

5년 차 개발자와 10년 차 개발자의 차이는 코딩 속도가 아닙니다. 같은 8시간을 일해도 결과물의 양과 질이 다른 이유는 일하는 방식의 차이에 있습니다. 제가 주니어 시절부터 시행착오를 겪으며 정립한 생산성 습관 7가지를 공유합니다.

## 1. 하루의 첫 30분은 계획에 투자한다

아침에 출근하자마자 코드를 치기 시작하면 안 됩니다. 그날 해야 할 일을 정리하고 우선순위를 매기는 데 30분을 투자하세요.

저는 매일 아침 세 가지를 적습니다.

- 오늘 반드시 끝내야 할 것 1가지
- 가능하면 하고 싶은 것 2가지
- 내일로 미룰 수 있는 것들

이렇게 하면 하루 종일 "다음에 뭘 하지?"라고 고민하는 시간이 사라집니다. 결정 피로를 줄이는 것만으로도 체감 생산성이 크게 올라갑니다.

## 2. 2시간 단위로 집중 블록을 만든다

인간의 집중력은 90~120분이 한계입니다. 이 시간 동안 슬랙 알림을 끄고, 이메일을 확인하지 않고, 오직 하나의 작업에만 집중합니다.

실천 방법은 간단합니다.

- 슬랙 상태를 "집중 모드"로 변경
- 브라우저에서 불필요한 탭 모두 닫기
- 음악이 필요하면 가사 없는 로파이나 백색소음
- 2시간 후 반드시 10~15분 휴식

이 방법을 쓰기 시작한 후 하루에 처리하는 PR 수가 눈에 띄게 늘었습니다.

## 3. 디버깅 시간을 기록한다

버그 하나에 3시간을 쓴 경험, 누구나 있을 겁니다. 문제는 그 3시간 동안 뭘 했는지 기억하지 못한다는 것입니다.

디버깅할 때 메모장을 열어두고 이렇게 기록합니다.

```
14:00 - 증상: 로그인 후 세션이 유지되지 않음
14:10 - 가설 1: 쿠키 설정 문제 → 확인 결과 쿠키는 정상
14:25 - 가설 2: 세션 만료 시간 → 서버 설정 확인, 30분으로 정상
14:40 - 가설 3: CORS 문제 → 개발자 도구에서 에러 발견!
14:50 - 해결: API 서버의 CORS 설정에 credentials 옵션 누락
```

이렇게 하면 같은 유형의 버그를 다시 만났을 때 빠르게 해결할 수 있고, 디버깅 능력 자체가 향상됩니다.

## 4. 코드 리뷰를 먼저 한다

PR 리뷰는 미루기 쉬운 일입니다. 하지만 내 PR이 리뷰되지 않아서 다음 작업을 못 하는 상황이 반복되면 팀 전체의 생산성이 떨어집니다.

저는 매일 오전 첫 집중 블록이 시작되기 전, 그리고 점심 식사 후에 팀원들의 PR을 리뷰합니다. 보통 한 번에 15~30분 정도 투자하면 충분합니다.

좋은 코드 리뷰의 핵심은 다음과 같습니다.

- 로직의 정확성보다 설계의 방향성에 집중
- "이렇게 하면 어떨까요?" 형태의 제안
- 잘된 부분에 대한 칭찬도 빠뜨리지 않기

## 5. 반복 작업은 즉시 자동화한다

세 번 이상 반복하는 작업이 있다면, 그 즉시 스크립트나 단축키로 자동화합니다.

대표적인 예시들입니다.

- **테스트 데이터 생성**: 매번 수동으로 더미 데이터를 만드는 대신 시드 스크립트 작성
- **배포 과정**: 여러 단계를 거치는 배포를 하나의 명령어로 통합
- **코드 스니펫**: 자주 쓰는 패턴을 VS Code 스니펫으로 등록
- **Git alias**: 자주 쓰는 git 명령어 단축

자동화에 투자하는 시간은 처음에 30분~1시간이지만, 앞으로 절약하는 시간은 수십 시간입니다.

## 6. 문서를 먼저 쓴다

코드를 치기 전에 문서를 먼저 씁니다. 거창한 문서가 아니라, 간단한 설계 메모입니다.

```markdown
## 기능: 비밀번호 재설정

### 플로우
1. 사용자가 이메일 입력
2. 서버에서 토큰 생성 후 이메일 발송
3. 사용자가 링크 클릭 → 토큰 검증
4. 새 비밀번호 입력 후 변경 완료

### 고려사항
- 토큰 유효기간: 30분
- 동일 이메일 재요청: 5분 쿨다운
- 비밀번호 규칙: 8자 이상, 대소문자+숫자

### API 엔드포인트
- POST /api/auth/forgot-password
- POST /api/auth/reset-password
```

이 과정에서 빠뜨린 케이스를 발견하거나, 더 나은 설계를 떠올리는 경우가 많습니다. 코딩하다가 "이거 어떻게 처리하지?"라고 멈추는 일이 줄어듭니다.

## 7. 퇴근 전 5분 회고

하루가 끝나기 전 5분 동안 짧은 회고를 합니다.

- 오늘 계획한 것 중 완료한 것은?
- 예상보다 오래 걸린 작업은? 왜?
- 내일 가장 먼저 해야 할 것은?

이 기록이 쌓이면 자신의 작업 패턴이 보이기 시작합니다. "화요일에 미팅이 많아서 집중 작업은 수요일에 몰아야겠다"와 같은 인사이트를 얻을 수 있습니다.

## 도구보다 습관

생산성 도구를 새로 찾아 헤매는 시간보다, 위의 습관 하나를 실천하는 것이 훨씬 효과적입니다. 모든 것을 한꺼번에 적용하려 하지 마세요. 하나씩 2주 동안 시도해보고, 본인에게 맞는 것을 남기세요. 작은 습관의 복리 효과는 생각보다 큽니다.
