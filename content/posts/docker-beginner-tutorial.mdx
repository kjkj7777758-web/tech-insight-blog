## Docker란 무엇인가?

### Docker의 개념

Docker는 애플리케이션을 **컨테이너(Container)** 라는 독립적이고 일관된 환경에 패키징하는 오픈소스 플랫폼입니다. 2013년 Solomon Hykes가 개발했으며, 2025년 현재 클라우드 환경에서 가장 널리 사용되는 컨테이너 기술입니다.

컨테이너는 가상머신(Virtual Machine)과 비슷해 보이지만, 훨씬 가볍고 빠릅니다. 운영체제 전체를 포함하는 가상머신과 달리, 컨테이너는 애플리케이션과 그 의존성만 포함합니다.

### Docker가 필요한 이유

**"내 컴퓨터에서는 잘 되는데, 다른 환경에서는 안 된다"**

이는 개발자들이 자주 겪는 문제입니다. 환경 차이로 인한 문제를 완벽하게 해결하는 것이 Docker의 목표입니다.

**Docker의 장점:**
- **일관성**: 개발 환경, 테스트 환경, 프로덕션 환경이 동일합니다
- **효율성**: 가상머신보다 가볍고 빠릅니다
- **이식성**: 모든 운영체제에서 동일하게 작동합니다
- **확장성**: 마이크로서비스 아키텍처에 최적입니다
- **자동화**: CI/CD 파이프라인을 쉽게 구축할 수 있습니다

## Docker의 기본 개념

### 1. 이미지 (Image)

이미지는 애플리케이션을 실행하기 위한 모든 것을 포함하는 **템플릿**입니다:
- 운영체제의 기본 계층
- 런타임 (Python, Node.js 등)
- 애플리케이션 코드
- 의존성 (라이브러리, 패키지 등)
- 환경 변수 및 설정

### 2. 컨테이너 (Container)

컨테이너는 이미지를 **실행한 인스턴스**입니다. 여러 개의 컨테이너가 동일한 이미지로부터 생성될 수 있으며, 각 컨테이너는 독립적으로 실행됩니다.

### 3. Dockerfile

Dockerfile은 이미지를 구축하기 위한 **설계도**입니다. Docker는 이 파일의 명령어들을 순차적으로 실행하여 이미지를 만듭니다.

### 4. 레지스트리 (Registry)

Docker Hub와 같은 레지스트리는 이미지를 저장하고 공유하는 저장소입니다. 마치 GitHub가 코드의 레지스트리이듯, Docker Hub는 이미지의 레지스트리입니다.

## Docker 설치하기

### Windows와 Mac

**Docker Desktop** 설치:
1. https://www.docker.com/products/docker-desktop 방문
2. 운영체제에 맞는 버전 다운로드
3. 설치 파일 실행
4. 터미널에서 `docker --version` 입력하여 설치 확인

```bash
# 설치 확인
docker --version
# Docker version 26.1.0, build d260a54
```

### Linux

Ubuntu 예시:

```bash
# 기존 Docker 제거
sudo apt-get remove docker docker-engine docker.io containerd runc

# 새 Docker 설치
sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io

# 설치 확인
docker --version
```

### 설치 후 확인

```bash
# 간단한 테스트 이미지 실행
docker run hello-world

# 출력 예시:
# Hello from Docker!
# This message shows that your installation appears to be working correctly.
```

## Docker의 기본 명령어

### 이미지 관련 명령어

**이미지 검색하기**
```bash
docker search python
```

**이미지 다운로드하기**
```bash
docker pull python:3.12
```

**로컬 이미지 확인**
```bash
docker images

# 출력:
# REPOSITORY   TAG       IMAGE ID       CREATED       SIZE
# python       3.12      abcd1234      2 weeks ago    920MB
# ubuntu       latest    xyz9876       3 weeks ago    77.8MB
```

**이미지 삭제**
```bash
docker rmi python:3.12
```

### 컨테이너 관련 명령어

**컨테이너 실행**
```bash
# 기본 실행
docker run ubuntu:latest

# 상호작용 모드로 실행
docker run -it ubuntu:latest /bin/bash

# 백그라운드에서 실행
docker run -d nginx:latest

# 포트 매핑과 함께 실행
docker run -p 8080:80 nginx:latest
```

**실행 중인 컨테이너 확인**
```bash
docker ps

# 모든 컨테이너 확인 (중지된 것도 포함)
docker ps -a
```

**컨테이너 중지**
```bash
docker stop container_id_or_name
```

**컨테이너 시작**
```bash
docker start container_id_or_name
```

**컨테이너 로그 확인**
```bash
docker logs container_id_or_name

# 실시간 로그 확인
docker logs -f container_id_or_name
```

**컨테이너 내부에서 명령 실행**
```bash
docker exec -it container_id_or_name /bin/bash
```

**컨테이너 삭제**
```bash
docker rm container_id_or_name

# 실행 중인 컨테이너 강제 삭제
docker rm -f container_id_or_name
```

## Dockerfile 작성하기

### Dockerfile 기본 구조

```dockerfile
# 기반 이미지 지정
FROM python:3.12-slim

# 작업 디렉토리 설정
WORKDIR /app

# 파일 복사
COPY requirements.txt .

# 의존성 설치
RUN pip install -r requirements.txt

# 애플리케이션 파일 복사
COPY . .

# 포트 노출
EXPOSE 5000

# 실행 명령
CMD ["python", "app.py"]
```

### Dockerfile의 주요 명령어

**FROM**: 기반이 될 이미지 지정
```dockerfile
FROM ubuntu:22.04
FROM python:3.12
FROM node:20-alpine
```

**RUN**: 이미지 구축 중에 명령 실행
```dockerfile
RUN apt-get update
RUN apt-get install -y curl
```

**COPY/ADD**: 호스트에서 이미지로 파일 복사
```dockerfile
COPY requirements.txt /app/
COPY ./src /app/src
```

**WORKDIR**: 컨테이너 내부의 작업 디렉토리 설정
```dockerfile
WORKDIR /app
```

**ENV**: 환경 변수 설정
```dockerfile
ENV PYTHON_VERSION=3.12
ENV DEBUG=false
```

**EXPOSE**: 노출할 포트 지정
```dockerfile
EXPOSE 3000
EXPOSE 8080
```

**CMD**: 컨테이너 실행 시 기본 명령
```dockerfile
CMD ["python", "app.py"]
CMD ["npm", "start"]
```

**ENTRYPOINT**: 컨테이너의 메인 프로세스 정의
```dockerfile
ENTRYPOINT ["python", "app.py"]
```

### 실전 예제: Python Flask 애플리케이션

**프로젝트 구조:**
```
my-flask-app/
├── Dockerfile
├── requirements.txt
├── app.py
└── templates/
    └── index.html
```

**requirements.txt:**
```
Flask==3.0.0
```

**app.py:**
```python
from flask import Flask

app = Flask(__name__)

@app.route('/')
def hello():
    return 'Hello, Docker!'

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000)
```

**Dockerfile:**
```dockerfile
# Python 3.12 기반 이미지
FROM python:3.12-slim

# 작업 디렉토리 설정
WORKDIR /app

# 의존성 복사 및 설치
COPY requirements.txt .
RUN pip install -r requirements.txt

# 애플리케이션 코드 복사
COPY . .

# 포트 노출
EXPOSE 5000

# 애플리케이션 실행
CMD ["python", "app.py"]
```

**빌드 및 실행:**
```bash
# 이미지 구축
docker build -t my-flask-app:1.0 .

# 컨테이너 실행
docker run -p 5000:5000 my-flask-app:1.0

# http://localhost:5000 접속하여 확인
```

## Docker Compose 사용하기

### Docker Compose란

복잡한 애플리케이션은 여러 개의 컨테이너로 구성됩니다 (웹 서버, 데이터베이스, 캐시 등). Docker Compose는 이러한 여러 컨테이너를 한 번에 관리하는 도구입니다.

### docker-compose.yml 작성

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "5000:5000"
    environment:
      - DATABASE_URL=postgresql://user:password@db:5432/mydb
    depends_on:
      - db
    volumes:
      - ./app:/app

  db:
    image: postgres:15
    environment:
      - POSTGRES_USER=user
      - POSTGRES_PASSWORD=password
      - POSTGRES_DB=mydb
    volumes:
      - postgres_data:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  postgres_data:
```

### Docker Compose 명령어

**모든 서비스 시작:**
```bash
docker-compose up
```

**백그라운드에서 실행:**
```bash
docker-compose up -d
```

**서비스 중지:**
```bash
docker-compose down
```

**로그 확인:**
```bash
docker-compose logs -f web
```

**특정 서비스만 재시작:**
```bash
docker-compose restart db
```

## Docker의 모범 사례

### 1. 작은 기본 이미지 사용

```dockerfile
# 권장: Alpine 기반 (약 50MB)
FROM python:3.12-alpine

# 비권장: 일반 Python 이미지 (약 920MB)
FROM python:3.12
```

### 2. 멀티스테이지 빌드

```dockerfile
# 빌드 스테이지
FROM python:3.12 as builder
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt

# 실행 스테이지
FROM python:3.12-slim
WORKDIR /app
COPY --from=builder /usr/local/lib/python3.12/site-packages /usr/local/lib/python3.12/site-packages
COPY . .
CMD ["python", "app.py"]
```

### 3. 레이어 캐싱 활용

```dockerfile
# 자주 변경되는 것을 나중에 배치
FROM python:3.12-slim
WORKDIR /app

# 의존성은 먼저 (변경 빈도 낮음)
COPY requirements.txt .
RUN pip install -r requirements.txt

# 코드는 나중에 (변경 빈도 높음)
COPY . .

CMD ["python", "app.py"]
```

### 4. 적절한 권한 설정

```dockerfile
# root가 아닌 사용자로 실행
RUN useradd -m appuser
USER appuser

COPY --chown=appuser:appuser . .
```

### 5. 헬스체크 추가

```dockerfile
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:5000/ || exit 1
```

## 2025년 Docker 활용 팁

### 1. Docker Hub 계정 활용

```bash
# 로그인
docker login

# 이미지 태그 지정
docker tag my-app:latest username/my-app:latest

# 이미지 푸시
docker push username/my-app:latest
```

### 2. CI/CD 통합

GitHub Actions, GitLab CI 등을 통해 자동으로 Docker 이미지를 빌드하고 배포할 수 있습니다.

### 3. 보안 고려사항

- 민감한 정보 (API 키, 비밀번호)는 환경 변수로 전달
- 공식 이미지 사용 (검증됨)
- 정기적인 이미지 업데이트
