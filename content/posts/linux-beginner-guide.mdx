## 왜 개발자에게 리눅스가 중요한가

웹 서버의 약 80% 이상이 리눅스 기반으로 운영되고 있습니다. AWS, Google Cloud, Azure 등 주요 클라우드 서비스에서 사용하는 서버도 대부분 리눅스입니다. 따라서 개발자라면 리눅스에 대한 기본적인 이해가 필수입니다.

리눅스를 배우면 다음과 같은 이점이 있습니다.

- 서버 관리 및 배포 역량 향상
- Docker, Kubernetes 등 컨테이너 기술 활용
- 셸 스크립트를 통한 업무 자동화
- 오픈소스 생태계에 대한 이해
- 시스템 프로그래밍 역량 강화

## 리눅스 배포판 선택

리눅스에는 수백 가지의 배포판(distribution)이 있습니다. 입문자에게 추천하는 배포판은 다음과 같습니다.

### Ubuntu (우분투)

가장 인기 있는 배포판으로, 커뮤니티가 크고 문서가 풍부합니다. LTS(Long Term Support) 버전은 5년간 보안 업데이트를 제공하므로, 안정적인 환경을 원하는 경우 LTS 버전을 선택하세요. 이 글에서는 우분투를 기준으로 설명합니다.

### Fedora

Red Hat 계열의 배포판으로, 최신 기술을 빠르게 도입합니다. RHEL(Red Hat Enterprise Linux) 기반 서버를 관리할 계획이라면 좋은 선택입니다.

### Linux Mint

우분투 기반으로, 윈도우에서 전환하는 사용자에게 친숙한 인터페이스를 제공합니다.

## 우분투 설치 방법

### 방법 1: WSL (Windows Subsystem for Linux)

윈도우 사용자라면 WSL이 가장 간편한 방법입니다. 별도의 파티션이나 가상 머신 없이 윈도우 안에서 리눅스를 사용할 수 있습니다.

```bash
# PowerShell을 관리자 권한으로 실행 후
wsl --install

# 특정 배포판 설치
wsl --install -d Ubuntu-22.04
```

설치 후 재부팅하면 우분투 터미널을 바로 사용할 수 있습니다. VS Code와도 완벽하게 통합되어, WSL 환경에서 직접 개발할 수 있습니다.

### 방법 2: 가상 머신 (VirtualBox)

VirtualBox를 사용하면 기존 운영체제 위에서 리눅스를 실행할 수 있습니다. 실제 리눅스 데스크톱 환경을 경험하고 싶다면 이 방법을 추천합니다.

### 방법 3: 듀얼 부팅

윈도우와 우분투를 함께 설치하여, 부팅 시 선택하는 방식입니다. 최고의 성능을 발휘하지만, 파티션 관리에 주의가 필요합니다.

## 필수 터미널 명령어

리눅스를 사용하려면 터미널 명령어에 익숙해져야 합니다. 가장 기본적이고 자주 사용하는 명령어를 알아보겠습니다.

### 파일 시스템 탐색

```bash
# 현재 디렉토리 확인
pwd

# 디렉토리 이동
cd /home/user/projects
cd ..         # 상위 디렉토리
cd ~          # 홈 디렉토리

# 파일 및 디렉토리 목록
ls            # 기본 목록
ls -la        # 상세 목록 (숨김 파일 포함)
ls -lh        # 파일 크기를 읽기 쉽게 표시
```

### 파일 및 디렉토리 관리

```bash
# 디렉토리 생성
mkdir my-project
mkdir -p parent/child/grandchild  # 중첩 디렉토리 생성

# 파일 생성 및 편집
touch newfile.txt         # 빈 파일 생성
nano newfile.txt          # nano 에디터로 편집
vim newfile.txt           # vim 에디터로 편집

# 복사, 이동, 삭제
cp file.txt backup.txt         # 파일 복사
cp -r folder1 folder2          # 디렉토리 복사
mv old.txt new.txt             # 이름 변경 / 이동
rm file.txt                    # 파일 삭제
rm -rf directory               # 디렉토리 강제 삭제 (주의!)
```

### 파일 내용 확인

```bash
cat file.txt              # 전체 내용 출력
head -n 20 file.txt       # 처음 20줄
tail -n 20 file.txt       # 마지막 20줄
tail -f log.txt           # 실시간 로그 모니터링
less file.txt             # 페이지 단위로 보기
grep "검색어" file.txt    # 텍스트 검색
```

### 권한 관리

```bash
# 파일 권한 확인
ls -l file.txt
# 출력: -rw-r--r-- 1 user group 1234 Nov 15 10:00 file.txt

# 권한 변경
chmod 755 script.sh       # rwxr-xr-x
chmod +x script.sh        # 실행 권한 추가

# 소유자 변경
chown user:group file.txt
```

### 시스템 관리

```bash
# 시스템 정보
uname -a                  # 커널 정보
df -h                     # 디스크 사용량
free -h                   # 메모리 사용량
top                       # 프로세스 모니터링

# 프로세스 관리
ps aux                    # 모든 프로세스 목록
kill PID                  # 프로세스 종료
```

## 패키지 관리

우분투는 APT(Advanced Package Tool)를 사용하여 소프트웨어를 설치합니다.

```bash
# 패키지 목록 업데이트
sudo apt update

# 시스템 업그레이드
sudo apt upgrade

# 패키지 설치
sudo apt install package-name

# 패키지 삭제
sudo apt remove package-name

# 패키지 검색
apt search keyword
```

## 개발 환경 구축

### Git 설치 및 설정

```bash
sudo apt install git

git config --global user.name "이름"
git config --global user.email "email@example.com"
```

### Node.js 설치 (NVM 사용)

```bash
# NVM 설치
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# 터미널 재시작 후
nvm install --lts
nvm use --lts

# 확인
node --version
npm --version
```

### Python 설치

```bash
# Python 3는 대부분 기본 설치되어 있음
python3 --version

# pip 설치
sudo apt install python3-pip

# 가상 환경 생성
python3 -m venv myenv
source myenv/bin/activate
```

### VS Code 연동

VS Code에서 Remote - WSL 확장을 설치하면 WSL 환경에서 직접 코드를 편집하고 실행할 수 있습니다. 터미널도 WSL 셸이 기본으로 열리므로 매끄러운 개발 경험을 제공합니다.

## 유용한 셸 기능

### 파이프와 리디렉션

```bash
# 파이프: 명령어 출력을 다음 명령어의 입력으로
ls -la | grep ".txt"
cat access.log | sort | uniq -c | sort -rn | head -10

# 리디렉션: 출력을 파일로 저장
echo "hello" > file.txt      # 덮어쓰기
echo "world" >> file.txt     # 추가
```

### 환경 변수

```bash
# 환경 변수 설정
export MY_VAR="hello"

# 영구 설정 (~/.bashrc에 추가)
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

### 별칭 (Alias)

자주 사용하는 긴 명령어를 짧게 만들 수 있습니다.

```bash
# ~/.bashrc에 추가
alias ll='ls -la'
alias gs='git status'
alias gp='git push'
alias dc='docker-compose'
```

## 다음 단계 추천

리눅스 기본기를 익혔다면, 다음 주제를 학습하는 것을 추천합니다.

2. **Docker**: 컨테이너 기반 개발 환경 구축
3. **서버 관리**: Nginx, systemd를 활용한 웹 서버 운영
4. **네트워킹**: SSH, 방화벽(UFW), 네트워크 진단 도구
5. **자동화**: cron을 활용한 스케줄링, CI/CD 파이프라인
