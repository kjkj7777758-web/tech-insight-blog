## Next.js란

Next.js는 React를 기반으로 한 풀스택 웹 프레임워크입니다. Vercel에서 개발하고 있으며, 서버 사이드 렌더링(SSR), 정적 사이트 생성(SSG), API 라우트 등 현대 웹 개발에 필요한 다양한 기능을 기본으로 제공합니다.

블로그를 만들기에 Next.js가 특히 적합한 이유는 다음과 같습니다.

- **뛰어난 SEO 지원**: 서버 사이드 렌더링과 정적 생성 덕분에 검색 엔진 최적화에 유리합니다.
- **빠른 페이지 로딩**: 자동 코드 분할과 이미지 최적화로 빠른 사용자 경험을 제공합니다.
- **파일 기반 라우팅**: 폴더 구조만으로 URL 경로가 자동으로 생성되어 직관적입니다.
- **간편한 배포**: Vercel을 통해 Git 저장소 연결만으로 자동 배포가 가능합니다.

## 프로젝트 생성

터미널을 열고 다음 명령어를 실행하여 새로운 Next.js 프로젝트를 생성합니다.

```tsx
// 터미널에서 실행
// npx create-next-app@latest my-blog

// 설치 중 아래 옵션을 선택합니다
// TypeScript: Yes
// ESLint: Yes
// Tailwind CSS: Yes
// src/ directory: Yes
// App Router: Yes
// import alias: @/*
```

프로젝트가 생성되면 해당 디렉토리로 이동하여 개발 서버를 실행합니다.

```tsx
// 터미널에서 실행
// cd my-blog
// npm run dev
```

브라우저에서 `http://localhost:3000`에 접속하면 기본 페이지가 표시됩니다.

## 폴더 구조 이해

Next.js 13 이상의 App Router를 사용하면 다음과 같은 폴더 구조를 갖게 됩니다.

```
my-blog/
├── src/
│   ├── app/
│   │   ├── layout.tsx      # 전체 레이아웃
│   │   ├── page.tsx         # 메인 페이지
│   │   ├── globals.css      # 전역 스타일
│   │   └── posts/
│   │       └── [slug]/
│   │           └── page.tsx # 개별 포스트 페이지
│   ├── components/          # 재사용 컴포넌트
│   └── lib/                 # 유틸리티 함수
├── content/                 # 블로그 포스트 파일
├── public/                  # 정적 파일
├── tailwind.config.ts       # Tailwind 설정
└── package.json
```

**app 디렉토리**가 핵심이며, 이 안의 폴더 구조가 곧 URL 경로가 됩니다. 예를 들어 `app/about/page.tsx` 파일을 만들면 `/about` 경로가 자동으로 생성됩니다.

## 페이지 라우팅

Next.js의 App Router에서는 `page.tsx` 파일이 해당 경로의 페이지 역할을 합니다. 메인 페이지를 수정해 봅시다.

```tsx
// src/app/page.tsx
export default function Home() {
  return (
    <main className="max-w-4xl mx-auto px-4 py-8">
      <h1 className="text-4xl font-bold mb-6">나의 기술 블로그</h1>
      <p className="text-lg text-gray-600">
        웹 개발과 프로그래밍에 대한 이야기를 나눕니다.
      </p>
    </main>
  );
}
```

## Tailwind CSS로 스타일링

Tailwind CSS는 유틸리티 기반의 CSS 프레임워크로, 클래스명만으로 스타일을 빠르게 적용할 수 있습니다. 프로젝트 생성 시 이미 설치되어 있으므로 바로 사용할 수 있습니다.

### 블로그 카드 컴포넌트 만들기

```tsx
// src/components/PostCard.tsx
import Link from "next/link";

interface PostCardProps {
  title: string;
  description: string;
  date: string;
  slug: string;
}

export default function PostCard({ title, description, date, slug }: PostCardProps) {
  return (
    <article className="border border-gray-200 rounded-lg p-6 hover:shadow-lg transition-shadow">
      <time className="text-sm text-gray-500">{date}</time>
      <h2 className="text-xl font-semibold mt-2 mb-3">
        <Link href={`/posts/${slug}`} className="hover:text-blue-600">
          {title}
        </Link>
      </h2>
      <p className="text-gray-600 line-clamp-2">{description}</p>
      <Link
        href={`/posts/${slug}`}
        className="inline-block mt-4 text-blue-600 hover:underline"
      >
        더 읽기
      </Link>
    </article>
  );
}
```

## 블로그 포스트 데이터 관리

블로그 포스트는 마크다운 파일로 관리하는 것이 편리합니다. `gray-matter` 패키지를 사용하여 마크다운 파일의 프론트매터를 파싱하고, `next-mdx-remote`를 사용하여 MDX 콘텐츠를 렌더링합니다.

```tsx
// src/lib/posts.ts
import fs from "fs";
import path from "path";
import matter from "gray-matter";

const postsDirectory = path.join(process.cwd(), "content");

export interface Post {
  slug: string;
  title: string;
  description: string;
  date: string;
  content: string;
}

export function getAllPosts(): Post[] {
  const fileNames = fs.readdirSync(postsDirectory);

  const posts = fileNames
    .filter((fileName) => fileName.endsWith(".mdx"))
    .map((fileName) => {
      const slug = fileName.replace(/\.mdx$/, "");
      const filePath = path.join(postsDirectory, fileName);
      const fileContents = fs.readFileSync(filePath, "utf8");
      const { data, content } = matter(fileContents);

      return {
        slug,
        title: data.title,
        description: data.description,
        date: data.date,
        content,
      };
    });

  return posts.sort((a, b) => (a.date > b.date ? -1 : 1));
}

export function getPostBySlug(slug: string): Post | undefined {
  const filePath = path.join(postsDirectory, `${slug}.mdx`);
  if (!fs.existsSync(filePath)) return undefined;

  const fileContents = fs.readFileSync(filePath, "utf8");
  const { data, content } = matter(fileContents);

  return {
    slug,
    title: data.title,
    description: data.description,
    date: data.date,
    content,
  };
}
```

## 동적 라우팅

개별 블로그 포스트 페이지를 동적 라우팅으로 구현합니다. `[slug]` 폴더를 사용하면 URL 파라미터를 받아 해당 포스트를 렌더링할 수 있습니다.

```tsx
// src/app/posts/[slug]/page.tsx
import { getPostBySlug, getAllPosts } from "@/lib/posts";
import { notFound } from "next/navigation";

interface PageProps {
  params: { slug: string };
}

export async function generateStaticParams() {
  const posts = getAllPosts();
  return posts.map((post) => ({
    slug: post.slug,
  }));
}

export default function PostPage({ params }: PageProps) {
  const post = getPostBySlug(params.slug);

  if (!post) {
    notFound();
  }

  return (
    <article className="max-w-3xl mx-auto px-4 py-8">
      <header className="mb-8">
        <time className="text-sm text-gray-500">{post.date}</time>
        <h1 className="text-3xl font-bold mt-2">{post.title}</h1>
        <p className="text-gray-600 mt-3">{post.description}</p>
      </header>
      <div className="prose prose-lg max-w-none">
        {/* MDX 콘텐츠가 여기에 렌더링됩니다 */}
      </div>
    </article>
  );
}
```

## Vercel 배포

Next.js 프로젝트를 배포하는 가장 간단한 방법은 **Vercel**을 사용하는 것입니다. Vercel은 Next.js를 만든 회사이므로 최적화된 배포 환경을 제공합니다.

### 배포 순서

1. 프로젝트를 GitHub 저장소에 푸시합니다.
2. [vercel.com](https://vercel.com)에 가입하고 GitHub 계정을 연동합니다.
3. "New Project"를 클릭하고 해당 저장소를 선택합니다.
4. 빌드 설정은 자동으로 감지되므로 "Deploy" 버튼을 클릭합니다.
5. 배포가 완료되면 자동으로 HTTPS 도메인이 부여됩니다.

이후 GitHub에 코드를 푸시할 때마다 **자동으로 재배포**가 이루어집니다. 또한 Pull Request를 생성하면 미리보기 배포(Preview Deployment)가 자동으로 생성되어 변경 사항을 확인할 수 있습니다.

## 마무리

이번 가이드에서는 Next.js를 사용하여 개인 블로그를 처음부터 만드는 과정을 살펴보았습니다. 프로젝트 생성부터 레이아웃 구성, 마크다운 기반 포스트 관리, 동적 라우팅, 그리고 Vercel 배포까지 전체 워크플로우를 다루었습니다.

추가로 고려할 수 있는 기능들은 다음과 같습니다.

- **다크 모드 지원**: `next-themes` 라이브러리를 활용하여 다크 모드를 구현할 수 있습니다.
- **검색 기능**: 포스트 제목과 내용을 기반으로 검색 기능을 추가할 수 있습니다.
- **RSS 피드**: 블로그 구독을 위한 RSS 피드를 자동 생성할 수 있습니다.
- **댓글 시스템**: Giscus 또는 Utterances를 연동하여 GitHub 기반 댓글 시스템을 도입할 수 있습니다.
- **SEO 최적화**: 메타 태그, Open Graph 이미지, 사이트맵 등을 설정하여 검색 노출을 개선할 수 있습니다.

Next.js는 지속적으로 발전하고 있는 프레임워크이므로 공식 문서를 참고하면서 최신 기능을 활용해 보시기 바랍니다.
