## Hooks란 무엇인가

React Hooks는 React 16.8에서 도입된 기능으로, 클래스 컴포넌트를 작성하지 않고도 **상태(state)** 와 **생명주기(lifecycle)** 기능을 함수 컴포넌트에서 사용할 수 있게 해주는 함수들입니다.

Hooks가 등장하기 전에는 상태 관리나 부수 효과(side effect) 처리를 위해 반드시 클래스 컴포넌트를 사용해야 했습니다. 이로 인해 코드가 복잡해지고, 로직을 재사용하기 어려운 문제가 있었습니다. Hooks는 이러한 문제를 해결하며 현재 React 개발의 표준으로 자리 잡았습니다.

### Hooks의 기본 규칙

Hooks를 사용할 때 반드시 지켜야 할 두 가지 규칙이 있습니다.

- **최상위에서만 호출**: 반복문, 조건문, 중첩 함수 안에서 Hooks를 호출하면 안 됩니다. 항상 컴포넌트 함수의 최상위 레벨에서 호출해야 합니다.
- **React 함수 안에서만 호출**: 일반 자바스크립트 함수에서는 Hooks를 호출할 수 없습니다. React 함수 컴포넌트 또는 커스텀 훅 안에서만 사용해야 합니다.

## useState 완벽 이해

`useState`는 가장 기본적인 훅으로, 함수 컴포넌트에 상태를 추가할 수 있게 해줍니다.

### 기본 사용법

```tsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>현재 카운트: {count}</p>
      <button onClick={() => setCount(count + 1)}>증가</button>
      <button onClick={() => setCount(count - 1)}>감소</button>
      <button onClick={() => setCount(0)}>초기화</button>
    </div>
  );
}
```

`useState`는 배열을 반환하며, 첫 번째 요소는 현재 상태값, 두 번째 요소는 상태를 변경하는 함수입니다.

### 객체 상태 관리

상태가 객체일 때는 스프레드 연산자를 사용하여 기존 값을 유지하면서 특정 필드만 변경해야 합니다.

```tsx
import { useState } from "react";

function UserForm() {
  const [user, setUser] = useState({
    name: "",
    email: "",
    age: 0,
  });

  const handleChange = (field: string, value: string | number) => {
    setUser((prev) => ({
      ...prev,
      [field]: value,
    }));
  };

  return (
    <form>
      <input
        value={user.name}
        onChange={(e) => handleChange("name", e.target.value)}
        placeholder="이름"
      />
      <input
        value={user.email}
        onChange={(e) => handleChange("email", e.target.value)}
        placeholder="이메일"
      />
    </form>
  );
}
```

### 함수형 업데이트

이전 상태값을 기반으로 새로운 상태를 계산할 때는 **함수형 업데이트** 패턴을 사용하는 것이 안전합니다.

```tsx
// 잘못된 방식: 비동기 업데이트 시 문제가 발생할 수 있음
setCount(count + 1);

// 올바른 방식: 이전 상태를 기반으로 업데이트
setCount((prevCount) => prevCount + 1);
```

## useEffect 활용법

`useEffect`는 컴포넌트에서 **부수 효과(side effect)** 를 수행할 때 사용합니다. API 호출, 이벤트 리스너 등록, 타이머 설정 등이 대표적인 부수 효과입니다.

### 기본 사용법

```tsx
import { useState, useEffect } from "react";

function UserProfile({ userId }: { userId: string }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    async function fetchUser() {
      setLoading(true);
      try {
        const response = await fetch(`/api/users/${userId}`);
        const data = await response.json();
        setUser(data);
      } catch (error) {
        console.error("사용자 정보를 불러오는 데 실패했습니다:", error);
      } finally {
        setLoading(false);
      }
    }

    fetchUser();
  }, [userId]); // userId가 변경될 때마다 실행

  if (loading) return <p>로딩 중...</p>;
  if (!user) return <p>사용자를 찾을 수 없습니다.</p>;

  return <div>{/* 사용자 정보 표시 */}</div>;
}
```

### 의존성 배열의 이해

`useEffect`의 두 번째 인자인 의존성 배열은 매우 중요한 역할을 합니다.

- **빈 배열 `[]`**: 컴포넌트 마운트 시 한 번만 실행됩니다.
- **값이 포함된 배열 `[value]`**: 배열 안의 값이 변경될 때마다 실행됩니다.
- **배열 생략**: 매 렌더링마다 실행됩니다. 성능 문제를 유발할 수 있으므로 주의해야 합니다.

### 클린업 함수

이벤트 리스너나 타이머를 등록한 경우, 컴포넌트가 언마운트될 때 정리해야 메모리 누수를 방지할 수 있습니다.

```tsx
useEffect(() => {
  const handleResize = () => {
    console.log("창 크기가 변경되었습니다:", window.innerWidth);
  };

  window.addEventListener("resize", handleResize);

  // 클린업 함수: 컴포넌트 언마운트 시 실행
  return () => {
    window.removeEventListener("resize", handleResize);
  };
}, []);
```

## useContext로 전역 상태 관리

`useContext`는 React의 Context API와 함께 사용하여 **prop drilling 없이** 컴포넌트 트리 전체에 데이터를 전달할 수 있게 해줍니다.

```tsx
import { createContext, useContext, useState, ReactNode } from "react";

// 1. Context 생성
interface ThemeContextType {
  theme: "light" | "dark";
  toggleTheme: () => void;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

// 2. Provider 컴포넌트 생성
export function ThemeProvider({ children }: { children: ReactNode }) {
  const [theme, setTheme] = useState<"light" | "dark">("light");

  const toggleTheme = () => {
    setTheme((prev) => (prev === "light" ? "dark" : "light"));
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}

// 3. 커스텀 훅으로 사용 편의성 향상
export function useTheme() {
  const context = useContext(ThemeContext);
  if (!context) {
    throw new Error("useTheme은 ThemeProvider 안에서 사용해야 합니다.");
  }
  return context;
}

// 4. 사용 예시
function ThemeToggleButton() {
  const { theme, toggleTheme } = useTheme();

  return (
    <button onClick={toggleTheme}>
      현재 테마: {theme === "light" ? "라이트" : "다크"} 모드
    </button>
  );
}
```

## useRef 사용법

`useRef`는 두 가지 주요 용도로 사용됩니다. 첫째는 **DOM 요소에 직접 접근**하는 것이고, 둘째는 **렌더링을 트리거하지 않는 값을 저장**하는 것입니다.

```tsx
import { useRef, useEffect } from "react";

function AutoFocusInput() {
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    // 컴포넌트 마운트 시 자동으로 포커스
    inputRef.current?.focus();
  }, []);

  return <input ref={inputRef} placeholder="자동으로 포커스됩니다" />;
}

// 렌더링 횟수 추적 예시
function RenderCounter() {
  const renderCount = useRef(0);

  useEffect(() => {
    renderCount.current += 1;
  });

  return <p>이 컴포넌트는 {renderCount.current}번 렌더링되었습니다.</p>;
}
```

`useRef`로 생성한 값은 변경되어도 컴포넌트가 다시 렌더링되지 않는다는 점이 `useState`와의 핵심 차이입니다.

## useMemo와 useCallback

### useMemo

`useMemo`는 **계산 비용이 높은 값을 메모이제이션**하여 불필요한 재계산을 방지합니다.

```tsx
import { useMemo, useState } from "react";

function ExpensiveList({ items, filter }: { items: string[]; filter: string }) {
  const filteredItems = useMemo(() => {
    console.log("필터링 연산 수행 중...");
    return items.filter((item) =>
      item.toLowerCase().includes(filter.toLowerCase())
    );
  }, [items, filter]);

  return (
    <ul>
      {filteredItems.map((item, index) => (
        <li key={index}>{item}</li>
      ))}
    </ul>
  );
}
```

### useCallback

`useCallback`은 **함수를 메모이제이션**하여 자식 컴포넌트의 불필요한 리렌더링을 방지합니다.

```tsx
import { useCallback, useState } from "react";

function TodoApp() {
  const [todos, setTodos] = useState<string[]>([]);

  const addTodo = useCallback((text: string) => {
    setTodos((prev) => [...prev, text]);
  }, []);

  const removeTodo = useCallback((index: number) => {
    setTodos((prev) => prev.filter((_, i) => i !== index));
  }, []);

  return (
    <div>
      <TodoInput onAdd={addTodo} />
      <TodoList todos={todos} onRemove={removeTodo} />
    </div>
  );
}
```

단, `useMemo`와 `useCallback`은 무조건 사용한다고 좋은 것이 아닙니다. 메모이제이션 자체에도 비용이 발생하므로, **실제로 성능 문제가 확인된 경우**에만 적용하는 것이 바람직합니다.

## 커스텀 훅 만들기

커스텀 훅을 사용하면 **반복되는 로직을 재사용 가능한 함수로 추출**할 수 있습니다. 커스텀 훅의 이름은 반드시 `use`로 시작해야 합니다.

### 로컬 스토리지 훅

```tsx
import { useState, useEffect } from "react";

function useLocalStorage<T>(key: string, initialValue: T) {
  const [storedValue, setStoredValue] = useState<T>(() => {
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch (error) {
      console.error("로컬 스토리지 읽기 오류:", error);
      return initialValue;
    }
  });

  useEffect(() => {
    try {
      window.localStorage.setItem(key, JSON.stringify(storedValue));
    } catch (error) {
      console.error("로컬 스토리지 쓰기 오류:", error);
    }
  }, [key, storedValue]);

  return [storedValue, setStoredValue] as const;
}

// 사용 예시
function Settings() {
  const [language, setLanguage] = useLocalStorage("language", "ko");
  const [fontSize, setFontSize] = useLocalStorage("fontSize", 16);

  return (
    <div>
      <select value={language} onChange={(e) => setLanguage(e.target.value)}>
        <option value="ko">한국어</option>
        <option value="en">English</option>
      </select>
    </div>
  );
}
```

### API 호출 훅

```tsx
import { useState, useEffect } from "react";

interface UseFetchResult<T> {
  data: T | null;
  loading: boolean;
  error: string | null;
}

function useFetch<T>(url: string): UseFetchResult<T> {
  const [data, setData] = useState<T | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    const controller = new AbortController();

    async function fetchData() {
      setLoading(true);
      setError(null);
      try {
        const response = await fetch(url, { signal: controller.signal });
        if (!response.ok) throw new Error("네트워크 응답이 정상이 아닙니다.");
        const result = await response.json();
        setData(result);
      } catch (err) {
        if (err instanceof Error && err.name !== "AbortError") {
          setError(err.message);
        }
      } finally {
        setLoading(false);
      }
    }

    fetchData();
    return () => controller.abort();
  }, [url]);

  return { data, loading, error };
}
```

## 자주 하는 실수와 해결법

### 1. 무한 루프

`useEffect` 안에서 상태를 변경할 때 의존성 배열을 잘못 설정하면 무한 루프가 발생할 수 있습니다.

```tsx
// 잘못된 예시: 무한 루프 발생
useEffect(() => {
  setCount(count + 1); // 상태 변경 -> 리렌더링 -> useEffect 재실행
}); // 의존성 배열 없음

// 올바른 예시
useEffect(() => {
  setCount((prev) => prev + 1);
}, []); // 마운트 시 한 번만 실행
```

### 2. 오래된 클로저 문제

이벤트 핸들러나 타이머 내부에서 상태를 참조할 때 오래된 값이 참조되는 문제가 발생할 수 있습니다. 함수형 업데이트나 `useRef`를 사용하여 해결합니다.

### 3. 의존성 배열 누락

ESLint의 `react-hooks/exhaustive-deps` 규칙을 활성화하면 누락된 의존성을 자동으로 감지하여 경고해 줍니다. 이 규칙을 반드시 활용하시기 바랍니다.

## 정리

React Hooks는 함수 컴포넌트의 기능을 크게 확장하여 더 간결하고 재사용 가능한 코드를 작성할 수 있게 해줍니다. 이 글에서 다룬 핵심 내용을 요약하면 다음과 같습니다.

- **useState**: 컴포넌트의 상태를 관리하는 가장 기본적인 훅입니다.
- **useEffect**: 부수 효과를 처리하며, 클린업 함수로 자원을 정리합니다.
- **useContext**: Context API와 연계하여 전역 상태를 관리합니다.
- **useRef**: DOM 접근 및 렌더링과 무관한 값을 저장합니다.
- **useMemo / useCallback**: 성능 최적화를 위한 메모이제이션 훅입니다.
- **커스텀 훅**: 반복되는 로직을 재사용 가능한 형태로 추출합니다.

실제 프로젝트에서 다양한 훅을 활용하면서 각 훅의 특성과 적절한 사용 시점을 익히는 것이 중요합니다.
