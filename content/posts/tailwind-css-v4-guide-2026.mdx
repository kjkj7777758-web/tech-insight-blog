## Tailwind CSS v4, 무엇이 달라졌나

2024년 말 발표된 **Tailwind CSS v4**는 단순한 버전 업그레이드가 아닌, **완전히 새로운 엔진 위에서 재설계**된 프레임워크입니다. 기존 JavaScript 기반 빌드 시스템을 버리고 **Rust 기반의 Oxide 엔진**을 도입하면서 빌드 속도와 개발 경험 모두 크게 개선되었습니다.

2026년 현재 대부분의 신규 프로젝트가 v4로 시작되고 있으며, 기존 v3 프로젝트의 마이그레이션도 활발히 진행 중입니다.

---

## 핵심 변경사항 7가지

### 1. CSS-first 설정 (tailwind.config.js 제거)

v4에서 가장 큰 변화는 **JavaScript 설정 파일이 사라진 것**입니다. 모든 설정을 CSS 파일 안에서 직접 합니다.

**v3 방식 (tailwind.config.js):**
```javascript
// tailwind.config.js
module.exports = {
  theme: {
    extend: {
      colors: {
        brand: '#3b82f6',
      },
      fontFamily: {
        sans: ['Pretendard', 'sans-serif'],
      },
    },
  },
}
```

**v4 방식 (CSS 파일):**
```css
/* app.css */
@import "tailwindcss";

@theme {
  --color-brand: #3b82f6;
  --font-sans: 'Pretendard', sans-serif;
}
```

`@theme` 지시어 안에 CSS 커스텀 속성으로 테마를 정의합니다. 별도의 설정 파일 없이 **CSS만으로 모든 것을 관리**할 수 있습니다.

### 2. 자동 컨텐츠 감지

v3에서는 `content` 배열에 파일 경로를 일일이 지정해야 했습니다.

```javascript
// v3: 수동으로 경로 지정
content: ['./src/**/*.{js,ts,jsx,tsx}', './components/**/*.{js,ts,jsx,tsx}']
```

v4에서는 **자동으로 프로젝트 파일을 감지**합니다. `.gitignore`에 포함되지 않은 모든 파일을 자동 스캔하므로, content 설정이 필요 없습니다.

특정 경로를 제외하고 싶다면:
```css
@import "tailwindcss";
@source not "./vendor";
```

### 3. Rust 기반 Oxide 엔진

새로운 Oxide 엔진은 Rust로 작성되어 **빌드 속도가 최대 10배 향상**되었습니다.

| 지표 | v3 | v4 | 개선 |
|------|-----|-----|------|
| 초기 빌드 | 350ms | 45ms | 7.8배 |
| 증분 빌드 | 190ms | 8ms | 23.8배 |
| 전체 재빌드 | 520ms | 65ms | 8배 |

특히 **증분 빌드(파일 하나 수정 시)**가 8ms로 거의 즉각적이어서, 저장하는 순간 브라우저에 반영됩니다.

### 4. 새로운 유틸리티 클래스

v4에서 추가된 주요 유틸리티 클래스들:

**컨테이너 쿼리:**
```html
<div class="@container">
  <div class="@sm:grid-cols-2 @lg:grid-cols-3">
    <!-- 부모 크기에 따라 반응 -->
  </div>
</div>
```

**3D 변환:**
```html
<div class="rotate-x-45 rotate-y-12 perspective-800">
  3D 효과 요소
</div>
```

**그라디언트 보간:**
```html
<div class="bg-linear-to-r from-blue-500 to-purple-500 interpolate-oklch">
  더 자연스러운 그라디언트
</div>
```

**필드 사이징:**
```html
<textarea class="field-sizing-content">
  내용에 맞게 자동 크기 조절
</textarea>
```

### 5. 변형(Variant) 통합 및 신규 추가

**새로운 변형:**
- `starting:` - `@starting-style` CSS 규칙
- `not-*:` - `:not()` 선택자
- `in-*:` - 부모 상태 기반 스타일링
- `nth-*:` - n번째 자식 선택

```html
<!-- 첫 번째와 마지막 제외한 요소에 border -->
<div class="not-first:not-last:border-t">...</div>

<!-- 3번째 요소마다 다른 배경색 -->
<div class="nth-3n:bg-gray-50">...</div>
```

### 6. @utility로 커스텀 유틸리티 정의

v3의 `@layer utilities` 대신 `@utility` 지시어를 사용합니다.

```css
@utility tab-4 {
  tab-size: 4;
}

@utility scrollbar-hidden {
  scrollbar-width: none;
  &::-webkit-scrollbar {
    display: none;
  }
}
```

이렇게 정의한 유틸리티는 `hover:`, `md:` 등 **모든 변형과 함께 사용**할 수 있습니다.

### 7. PostCSS 플러그인 없이 독립 실행

v4는 PostCSS 플러그인 없이 **독립 CLI로 실행**할 수 있습니다.

```bash
# 독립 실행 (PostCSS 불필요)
npx @tailwindcss/cli -i app.css -o output.css --watch
```

물론 기존처럼 PostCSS 플러그인이나 Vite 플러그인으로도 사용 가능합니다.

```bash
# Vite 플러그인
npm install @tailwindcss/vite
```

```typescript
// vite.config.ts
import tailwindcss from '@tailwindcss/vite'

export default defineConfig({
  plugins: [tailwindcss()],
})
```

## v3에서 v4로 마이그레이션

### 자동 마이그레이션 도구

Tailwind 팀에서 제공하는 공식 마이그레이션 도구를 사용하면 대부분의 변경을 자동으로 처리합니다.

```bash
npx @tailwindcss/upgrade
```

이 도구가 자동으로 처리하는 것들:
- `tailwind.config.js` → CSS `@theme` 변환
- 유틸리티 클래스명 변경 (예: `bg-opacity-50` → `bg-black/50`)
- PostCSS 설정 업데이트
- `@apply` 구문 호환성 처리

### 주요 Breaking Changes

| v3 | v4 | 변경 이유 |
|----|-----|----------|
| `bg-opacity-50` | `bg-black/50` | 색상 투명도를 슬래시로 통합 |
| `ring` | `ring-3` | 기본 ring 너비 명시 |
| `shadow` | `shadow-sm` | shadow 스케일 재조정 |
| `blur` | `blur-sm` | blur 스케일 재조정 |
| `rounded` | `rounded-sm` | 반올림 값 변경 |
| `border` | `border` (동일) | 기본 border color가 currentColor로 변경 |

### 단계별 마이그레이션 가이드

**1단계: 의존성 업데이트**
```bash
npm install tailwindcss@latest @tailwindcss/vite@latest
npm uninstall @tailwindcss/typography @tailwindcss/forms  # v4에 내장됨
```

**2단계: 자동 마이그레이션 실행**
```bash
npx @tailwindcss/upgrade
```

**3단계: CSS 진입점 업데이트**
```css
/* 기존 v3 */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* v4 */
@import "tailwindcss";
```

**4단계: 커스텀 테마 이전**
```css
@import "tailwindcss";

@theme {
  --color-primary: #3b82f6;
  --color-secondary: #10b981;
  --font-sans: 'Pretendard', system-ui, sans-serif;
  --breakpoint-xs: 480px;
}
```

**5단계: 빌드 확인 및 시각적 테스트**

마이그레이션 후 반드시 모든 페이지를 시각적으로 확인하세요. 특히 `shadow`, `rounded`, `blur` 등 크기가 변경된 유틸리티가 의도한 대로 표시되는지 체크합니다.

## v4에서 달라진 개발 워크플로우

### 1. 설정이 간결해졌다

```css
/* 이것만으로 Tailwind 시작 가능 */
@import "tailwindcss";
```

별도의 설정 파일, PostCSS 설정, content 경로 지정 없이 **CSS 한 줄**로 시작할 수 있습니다.

### 2. 다크 모드가 더 쉬워졌다

```css
@import "tailwindcss";

@theme {
  --color-bg: #ffffff;
  --color-text: #1a1a1a;
}

/* 다크 모드 테마 */
@theme dark {
  --color-bg: #0a0a0a;
  --color-text: #f5f5f5;
}
```

### 3. 디자인 토큰 관리

v4의 `@theme`은 CSS 커스텀 속성을 생성하므로, JavaScript에서도 직접 접근할 수 있습니다.

```typescript
// CSS 변수에서 직접 테마 값 읽기
const primaryColor = getComputedStyle(document.documentElement)
  .getPropertyValue('--color-primary');
```

## 실전 예제: 카드 컴포넌트

v4의 새 기능들을 활용한 카드 컴포넌트 예시:

```html
<div class="@container">
  <article class="group rounded-xl bg-white shadow-xs
    hover:shadow-md transition-shadow duration-200
    @sm:flex @sm:gap-4">

    <!-- 이미지 -->
    <img
      src="/thumbnail.jpg"
      alt="포스트 썸네일"
      class="aspect-video w-full rounded-t-xl object-cover
        @sm:w-48 @sm:rounded-l-xl @sm:rounded-t-none"
    />

    <!-- 콘텐츠 -->
    <div class="p-4 @sm:py-3">
      <span class="text-xs font-medium text-blue-600">가이드</span>
      <h3 class="mt-1 text-lg font-bold text-gray-900
        group-hover:text-blue-600 transition-colors">
        Tailwind CSS v4 가이드
      </h3>
      <p class="mt-2 text-sm text-gray-600 line-clamp-2">
        v4에서 달라진 점과 마이그레이션 전략을 소개합니다.
      </p>
      <time class="mt-3 block text-xs text-gray-400">
        2026.03.12
      </time>
    </div>
  </article>
</div>
```

**포인트:**
- `@container` + `@sm:` 컨테이너 쿼리로 부모 크기에 반응
- `group` + `group-hover:` 상위 요소 hover 시 하위 요소 스타일 변경
- `shadow-xs`, `rounded-xl` 등 v4 스케일 사용
- `line-clamp-2` 내장 유틸리티 (플러그인 불필요)

## 자주 묻는 질문 (FAQ)

### Q. 기존 v3 프로젝트를 당장 v4로 바꿔야 하나요?
**아닙니다.** v3는 여전히 안정적으로 동작합니다. 신규 프로젝트는 v4로 시작하되, 기존 프로젝트는 일정 여유가 있을 때 마이그레이션하는 것을 추천합니다.

### Q. Tailwind CSS v4는 어떤 프레임워크와 호환되나요?
**Next.js, Nuxt, SvelteKit, Remix, Astro, Vite** 등 모든 주요 프레임워크와 호환됩니다. 각 프레임워크에 맞는 설정 가이드가 공식 문서에 제공됩니다.

### Q. @apply는 v4에서도 사용 가능한가요?
**네, 사용 가능합니다.** 다만 Tailwind 팀은 `@apply` 사용을 최소화하고 유틸리티 클래스를 직접 사용하는 것을 권장합니다.

### Q. TypeScript 타입 지원은 어떤가요?
v4부터 CSS 변수 기반이므로 별도의 TypeScript 설정이 필요 없습니다. 에디터 자동완성은 **Tailwind CSS IntelliSense VS Code 확장**이 v4를 완벽 지원합니다.

### Q. Tailwind UI 컴포넌트는 v4와 호환되나요?
**네**, Tailwind UI는 v4 호환 버전이 이미 제공되고 있습니다. 기존 v3 컴포넌트도 자동 마이그레이션 도구로 변환할 수 있습니다.

## 마무리

Tailwind CSS v4는 **더 빠르고, 더 간결하고, 더 강력해졌습니다.** CSS-first 설정 방식은 진입 장벽을 낮추면서도 유연성을 높였고, Oxide 엔진은 대규모 프로젝트에서도 빌드 성능 걱정을 없애줍니다.

프론트엔드 개발자라면 v4의 새로운 기능들을 하나씩 익혀보는 것을 추천합니다. 컨테이너 쿼리, 3D 변환, 새로운 변형 등 v4에서만 가능한 기능들이 개발 생산성을 크게 높여줄 것입니다.
