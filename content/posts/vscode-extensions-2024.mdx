## 들어가며

Visual Studio Code는 현재 가장 많이 사용되는 코드 에디터로, 풍부한 확장 프로그램 생태계가 큰 강점입니다. 하지만 마켓플레이스에 수만 개의 확장이 등록되어 있어 어떤 것을 설치해야 할지 고민되는 경우가 많습니다. 이 글에서는 실제로 개발 생산성을 크게 높여주는 확장 프로그램 20가지를 카테고리별로 정리하여 소개합니다.

## 코드 품질 및 포매팅 도구

### 1. Prettier - Code Formatter

**Prettier**는 코드를 자동으로 정리해주는 포매터입니다. JavaScript, TypeScript, CSS, HTML, JSON, Markdown 등 다양한 언어를 지원합니다. 팀 프로젝트에서 일관된 코드 스타일을 유지하는 데 필수적이며, 저장 시 자동 포매팅 기능을 설정하면 별도의 작업 없이 깔끔한 코드를 유지할 수 있습니다.

### 2. ESLint

**ESLint**는 JavaScript와 TypeScript 코드에서 잠재적인 오류와 안티패턴을 감지하는 린터입니다. 코드를 작성하는 즉시 문제를 발견하고 수정할 수 있어 버그를 사전에 예방하는 데 큰 도움이 됩니다. Prettier와 함께 사용하면 코드 품질과 스타일을 동시에 관리할 수 있습니다.

### 3. Error Lens

**Error Lens**는 에러, 경고, 정보 메시지를 코드 옆에 인라인으로 표시해주는 확장입니다. 기본적으로 VS Code는 밑줄만 표시하고 마우스를 올려야 메시지를 확인할 수 있지만, Error Lens를 사용하면 문제를 한눈에 파악할 수 있습니다.

### 4. SonarLint

**SonarLint**는 코드 품질과 보안 취약점을 실시간으로 검사하는 도구입니다. 코드 스멜(Code Smell), 버그, 보안 이슈를 감지하며 해결 방법도 함께 제안해 줍니다. Java, Python, JavaScript, TypeScript 등 다양한 언어를 지원합니다.

## 버전 관리 도구

### 5. GitLens

**GitLens**는 VS Code의 Git 기능을 대폭 강화해주는 확장입니다. 코드 라인별로 마지막 수정자와 커밋 메시지를 표시하며, 파일 히스토리, 브랜치 비교, 커밋 그래프 등의 기능을 제공합니다. 팀 프로젝트에서 코드의 변경 이력을 추적하는 데 매우 유용합니다.

### 6. Git Graph

**Git Graph**는 Git 저장소의 커밋 히스토리를 시각적인 그래프로 보여주는 확장입니다. 브랜치의 흐름과 머지 상태를 직관적으로 파악할 수 있으며, 그래프에서 직접 체크아웃, 머지, 리베이스 등의 작업을 수행할 수 있습니다.

## 테마 및 아이콘

### 7. One Dark Pro

**One Dark Pro**는 Atom 에디터의 인기 테마를 VS Code에 그대로 구현한 다크 테마입니다. 눈의 피로를 줄이는 부드러운 색상 조합이 특징이며, 장시간 코딩에 적합합니다. 가장 많이 다운로드된 테마 중 하나로 꾸준한 인기를 얻고 있습니다.

### 8. Material Icon Theme

**Material Icon Theme**는 파일과 폴더에 직관적인 아이콘을 적용해주는 확장입니다. 파일 확장자와 폴더 이름에 따라 자동으로 적절한 아이콘이 표시되어 파일 탐색기에서 원하는 파일을 빠르게 찾을 수 있습니다. `components`, `utils`, `hooks` 등 특정 폴더명에 대한 전용 아이콘도 제공합니다.

### 9. Catppuccin Theme

**Catppuccin**은 따뜻하고 파스텔 톤의 색상을 사용하는 테마로, 최근 개발자들 사이에서 급격히 인기를 얻고 있습니다. Latte(라이트), Frappe, Macchiato, Mocha(다크) 등 네 가지 변형을 제공하여 취향에 맞게 선택할 수 있습니다.

## 언어별 추천 확장

### 10. Python (Microsoft)

**Python 확장**은 파이썬 개발에 필요한 핵심 기능을 모두 제공합니다. IntelliSense(코드 자동 완성), 디버깅, 린팅, 코드 네비게이션, Jupyter Notebook 지원 등이 포함되어 있습니다. 파이썬 개발자라면 반드시 설치해야 할 확장입니다.

### 11. ES7+ React/Redux/React-Native Snippets

**ES7+ Snippets**는 React, Redux, React Native 개발 시 자주 사용하는 코드 조각을 단축 명령어로 빠르게 생성해주는 확장입니다. 예를 들어 `rafce`를 입력하면 Arrow Function 컴포넌트가 자동으로 생성되어 반복적인 코드 작성 시간을 크게 줄일 수 있습니다.

### 12. Tailwind CSS IntelliSense

**Tailwind CSS IntelliSense**는 Tailwind CSS 클래스명에 대한 자동 완성, 구문 강조, 린팅 기능을 제공합니다. 클래스명 위에 마우스를 올리면 실제 CSS 속성을 미리 볼 수 있어 Tailwind CSS를 사용하는 프로젝트에서 생산성을 크게 향상시켜 줍니다.

### 13. Go (Google)

**Go 확장**은 Google에서 공식 지원하는 Go 언어 개발 도구입니다. 코드 자동 완성, 디버깅, 테스트 실행, 코드 포매팅, 리팩토링 등 Go 개발에 필요한 모든 기능을 제공합니다.

## AI 코딩 도우미

### 14. GitHub Copilot

**GitHub Copilot**은 AI 기반 코드 자동 완성 도구로, 작성 중인 코드의 맥락을 이해하고 다음에 올 코드를 예측하여 제안합니다. 반복적인 코드 작성을 줄이고, 새로운 라이브러리나 API의 사용법을 빠르게 파악하는 데 도움이 됩니다. 유료 서비스이지만 학생과 오픈소스 기여자에게는 무료로 제공됩니다.

### 15. Codeium

**Codeium**은 무료 AI 코딩 어시스턴트입니다. GitHub Copilot과 유사한 코드 자동 완성 기능을 제공하며, 70개 이상의 프로그래밍 언어를 지원합니다. 무료로 사용할 수 있다는 점이 가장 큰 장점이며, 개인 프로젝트나 학습 용도로 특히 유용합니다.

## 유용한 유틸리티

### 16. Live Server

**Live Server**는 정적 웹 페이지를 로컬 서버에서 실행하고, 파일을 저장할 때마다 자동으로 브라우저를 새로고침해주는 확장입니다. HTML, CSS, JavaScript를 학습하거나 간단한 웹 페이지를 개발할 때 매우 유용합니다.

### 17. Thunder Client

**Thunder Client**는 VS Code 안에서 REST API를 테스트할 수 있게 해주는 경량 HTTP 클라이언트입니다. Postman과 유사한 기능을 에디터 내에서 바로 사용할 수 있어 별도의 프로그램을 실행할 필요가 없습니다. 요청 컬렉션 관리, 환경 변수 설정 등의 기능도 지원합니다.

### 18. Auto Rename Tag

**Auto Rename Tag**는 HTML/JSX에서 여는 태그를 수정하면 닫는 태그도 자동으로 함께 변경해주는 확장입니다. 단순하지만 매일 사용하게 되는 매우 실용적인 도구입니다.

### 19. Path Intellisense

**Path Intellisense**는 파일 경로를 입력할 때 자동 완성을 제공합니다. import 구문이나 파일 참조 시 경로를 빠르고 정확하게 입력할 수 있어 오타로 인한 오류를 줄여줍니다.

### 20. Todo Tree

**Todo Tree**는 코드에 작성된 `TODO`, `FIXME`, `HACK` 등의 주석을 자동으로 탐지하여 사이드바에 트리 구조로 정리해줍니다. 프로젝트 전체에서 처리해야 할 작업을 한눈에 확인할 수 있어 대규모 프로젝트 관리에 특히 유용합니다.

## 추천 설정 팁

확장 프로그램을 설치한 후 다음 설정을 적용하면 더 효율적으로 사용할 수 있습니다.

### 저장 시 자동 포매팅

VS Code 설정(settings.json)에 다음 항목을 추가하면 파일 저장 시 자동으로 코드가 정리됩니다.

- **Format On Save**: 파일 저장 시 자동 포매팅을 활성화합니다.
- **Default Formatter**: 기본 포매터를 Prettier로 설정합니다.
- **Editor Tab Size**: 탭 크기를 2로 설정하면 코드가 더 간결해 보입니다.

### 프로젝트별 확장 추천

`.vscode/extensions.json` 파일을 생성하면 프로젝트에 필요한 확장을 팀원에게 자동으로 추천할 수 있습니다. 프로젝트를 처음 열 때 권장 확장 목록이 표시되어 개발 환경 설정 시간을 줄여줍니다.

### 단축키 커스터마이징

자주 사용하는 기능에 단축키를 할당하면 마우스 사용을 줄이고 작업 속도를 높일 수 있습니다. `Ctrl + K, Ctrl + S`로 단축키 설정 화면에 접근할 수 있습니다.

## 마무리

이 글에서 소개한 20가지 확장 프로그램은 모두 실제 개발 현장에서 검증된 도구들입니다. 모든 확장을 한꺼번에 설치하기보다는 자신의 개발 환경과 사용하는 기술 스택에 맞는 것을 골라 설치하는 것을 권장합니다.

확장 프로그램을 너무 많이 설치하면 에디터 성능이 저하될 수 있으므로 사용하지 않는 확장은 비활성화하거나 제거하는 것이 좋습니다. 또한 정기적으로 확장 목록을 검토하여 더 나은 대안이 있는지 확인하는 습관을 기르면 항상 최적의 개발 환경을 유지할 수 있습니다.
