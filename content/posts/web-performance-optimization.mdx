## 웹 성능이 중요한 이유

2025년, 웹사이트의 속도는 단순한 사용자 경험의 문제가 아닙니다. Google의 알고리즘은 페이지 속도를 검색 순위의 중요한 요소로 고려하고 있으며, 사용자들은 느린 사이트를 빠르게 떠나갑니다. 연구에 따르면 페이지 로드 시간이 1초 증가할 때마다 이탈률이 7% 증가합니다.

따라서 웹사이트 성능 최적화는 SEO, 사용자 만족도, 변환율 향상을 위해 필수적입니다. 이 가이드에서는 웹사이트 속도를 획기적으로 개선하는 실전 기법들을 상세히 설명합니다.

## Core Web Vitals: Google의 성능 기준

### Core Web Vitals란

Core Web Vitals는 Google이 정의한 웹 성능의 세 가지 핵심 지표입니다. 2021년부터 검색 순위에 영향을 미치기 시작했으며, 2025년에는 더욱 중요해졌습니다.

### 1. Largest Contentful Paint (LCP)

**정의**: 페이지가 로드될 때 가장 큰 콘텐츠 요소가 화면에 나타나는 시간입니다.

**목표**: 2.5초 이내

**개선 방법**:
- 이미지와 비디오 최적화 (아래에서 상세 설명)
- 서버 응답 시간 개선 (CDN 사용)
- CSS와 JavaScript 최적화
- 렌더링 차단 리소스 제거

**측정 방법**:
```javascript
// JavaScript로 LCP 측정하기
const observer = new PerformanceObserver((list) => {
  const entries = list.getEntries();
  const lastEntry = entries[entries.length - 1];
  console.log('LCP:', lastEntry.renderTime || lastEntry.loadTime);
});

observer.observe({entryTypes: ['largest-contentful-paint']});
```

### 2. First Input Delay (FID)

**정의**: 사용자가 페이지와 상호작용할 때부터 브라우저가 반응하는 데 걸리는 시간입니다.

**목표**: 100ms 이내

**개선 방법**:
- JavaScript 실행 시간 단축
- 메인 스레드 작업 최소화
- Web Worker 활용
- 써드파티 스크립트 최적화

### 3. Cumulative Layout Shift (CLS)

**정의**: 사용자가 예상하지 않은 레이아웃 변화의 총합입니다.

**목표**: 0.1 이하

**개선 방법**:
- 이미지와 비디오에 명시적 크기 지정
- 폰트 로딩 최적화
- 광고 공간 사전 할당
- 동적 콘텐츠 제어

```css
/* 레이아웃 시프트 방지 예제 */
img {
  width: 300px;
  height: 200px;
  aspect-ratio: 300 / 200;
}
```

## 이미지 최적화: 성능 개선의 핵심

### 이미지 형식 선택

웹에 사용되는 최적의 이미지 형식을 선택하는 것이 중요합니다.

**JPEG**: 사진과 같은 복잡한 이미지에 최적
- 크기가 작음
- 색상 손실 가능 (문제 없는 정도)
- 호환성 우수

**PNG**: 투명도가 필요할 때
- 무손실 압축
- 파일 크기가 상대적으로 큼
- 로고, 아이콘에 적합

**WebP**: 최신 형식, 최고의 압축
- JPEG보다 25-35% 작음
- 투명도 지원
- 오래된 브라우저에서는 미지원

**AVIF**: 2025년 최신 형식
- 가장 우수한 압축률
- WebP보다 20% 더 작음
- 아직 브라우저 지원이 제한적

### 반응형 이미지 구현

```html
<!-- 다양한 해상도에 대한 이미지 제공 -->
<picture>
  <source srcset="image.avif" type="image/avif">
  <source srcset="image.webp" type="image/webp">
  <source srcset="image.jpg" type="image/jpeg">
  <img src="image.jpg" alt="설명">
</picture>

<!-- srcset를 사용한 반응형 이미지 -->
<img
  srcset="small.jpg 480w, medium.jpg 800w, large.jpg 1200w"
  sizes="(max-width: 600px) 100vw, 50vw"
  src="medium.jpg"
  alt="설명"
>
```

### 이미지 압축

온라인 도구를 사용하거나 빌드 프로세스에 자동화를 추가하세요:

- **TinyPNG/TinyJPG**: 무손실 압축, 웹 기반
- **ImageOptim**: Mac용, 배치 처리
- **ImageMagick**: CLI 도구, 자동화에 최적

```bash
# ImageMagick으로 일괄 압축
convert input.jpg -quality 85 -strip output.jpg
```

## 지연 로딩 (Lazy Loading)

### 기본 Lazy Loading

```html
<!-- HTML5 loading 속성 -->
<img src="image.jpg" alt="설명" loading="lazy">

<!-- 자동으로 필요할 때만 로드됩니다 -->
<iframe src="video.html" loading="lazy"></iframe>
```

### 고급 Intersection Observer API

```javascript
// 더 세밀한 제어를 원할 때
const lazyImages = document.querySelectorAll('img[data-src]');

const imageObserver = new IntersectionObserver((entries, observer) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target;
      img.src = img.dataset.src;
      img.classList.add('loaded');
      observer.unobserve(img);
    }
  });
});

lazyImages.forEach(img => imageObserver.observe(img));
```

HTML:
```html
<img data-src="image.jpg" alt="설명" class="lazy">
```

## 캐싱 전략

### 브라우저 캐싱

```
Cache-Control: public, max-age=31536000

# 자주 변경되는 콘텐츠
Cache-Control: public, max-age=3600

# 매번 검증 필요
Cache-Control: no-cache
```

### 서비스 워커 캐싱

```javascript
// 네트워크 먼저, 캐시 폴백
self.addEventListener('fetch', (event) => {
  event.respondWith(
    fetch(event.request)
      .then(response => {
        // 네트워크 요청 성공
        const clone = response.clone();
        caches.open('v1').then(cache => {
          cache.put(event.request, clone);
        });
        return response;
      })
      .catch(() => {
        // 네트워크 실패, 캐시 사용
        return caches.match(event.request);
      })
  );
});
```

## JavaScript 최적화

### Code Splitting (코드 분할)

```javascript
// Webpack 설정
import React from 'react';
const HeavyComponent = React.lazy(() =>
  import('./HeavyComponent')
);

function App() {
  return (
    <React.Suspense fallback={<div>로딩 중...</div>}>
      <HeavyComponent />
    </React.Suspense>
  );
}
```

### 동적 Import

```javascript
// 필요할 때만 모듈 로드
button.addEventListener('click', async () => {
  const module = await import('./expensive-module.js');
  module.doSomething();
});
```

### 써드파티 스크립트 최적화

```html
<!-- 성능이 중요하지 않은 스크립트는 async 사용 -->
<script async src="analytics.js"></script>

<!-- 실행 순서가 중요하면 defer 사용 -->
<script defer src="main.js"></script>
```

## 성능 모니터링

### Google PageSpeed Insights

1. https://pagespeed.web.dev 방문
2. 분석할 URL 입력
3. Core Web Vitals 점수 확인
4. 개선 제안 검토

### Lighthouse 활용

Chrome DevTools의 Lighthouse 탭에서:
1. Performance 탭 선택
2. "Analyze page load" 클릭
3. 상세 보고서 검토
4. 우선순위별 개선사항 확인

```javascript
// JavaScript로 성능 측정
const perfData = window.performance.timing;
const pageLoadTime = perfData.loadEventEnd - perfData.navigationStart;
console.log('Page Load Time:', pageLoadTime + 'ms');
```

## 2025년 웹 성능 최적화의 트렌드

### 1. INP (Interaction to Next Paint)

FID를 대체하는 새로운 지표로, 모든 상호작용의 반응성을 측정합니다. 목표는 200ms 이내입니다.

### 2. Edge 컴퓨팅

Cloudflare Workers, Vercel Edge Functions 등을 통해 서버 응답 시간을 대폭 단축할 수 있습니다.

### 3. 자동 최적화 도구

Next.js, Gatsby 등 모던 프레임워크는 이미지와 코드 분할을 자동으로 최적화합니다.

## 결론

웹사이트 성능 최적화는 일회성 작업이 아닌 지속적인 과정입니다. Core Web Vitals 기준을 충족하고, 정기적으로 성능을 모니터링하며, 사용자의 피드백에 귀 기울이세요.

2025년, 다음 단계의 성능 최적화를 통해 사용자 만족도를 높이고, 검색 순위를 개선하고, 비즈니스 성과를 극대화하세요!
